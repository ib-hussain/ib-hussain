I went through your logs and screenshots carefully. The two hard failures are clear: your **bare Super is still bound to the wrong action**, and the **two local GNOME extensions were created on disk but never actually became enabled**. I built v11 to fix those first, then restore dock styling without going back to the broken Xorg path.

I kept **Dash-to-Panel** as the base because it is the GNOME extension already designed to combine the launcher/taskbar and status/tray area into one panel, and it supports the exact knobs you are already using such as panel-length percentages, element positioning, intellihide, and transparency. The tradeoff is that it still behaves like a panel internally, not a true content-width macOS dock. ([GitHub][1])

I also moved the clock and opacity work back into **proper local GNOME Shell extensions** instead of loose scripts, because GNOME expects extensions to live under `~/.local/share/gnome-shell/extensions/<uuid>/` with `metadata.json` and `extension.js`, and GNOME 45+ uses ES modules for extension code. ([gjs.guide][2])

One important constraint: I did **not** try to fake true backdrop blur, because GNOME Shell CSS does not support blur effects through CSS alone. The stable route here is semi-transparent glass styling with border/shadow, not actual macOS-style CSS blur. ([GNOME Discourse][3])

On power modes, your VM is still a hard limit. `powerprofilesctl` only reports profiles the platform actually exposes, so VirtualBox can leave it as `unavailable`, which is why you are not getting the full hardware-backed mode behaviour you want inside the guest. ([Arch Manual Pages][4])
