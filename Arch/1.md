Yes. For your exact layout, I would **not** use Enlightenment’s default shelf as the main taskbar. It is too limited for the Windows/macOS hybrid dock you described. I would use:

* **Enlightenment** = desktop/window manager
* **Eww** = custom bottom dock and action centre
* **Rofi** = app launcher/search
* **sxhkd + xcape** = make the Super/Windows key open the launcher
* **Nemo** = Windows-like file manager option
* **PipeWire + pamixer + pavucontrol** = sound widget backend
* **NetworkManager CLI** = network/status backend
* **cpupower / power-profiles-daemon** = performance mode backend

Eww is a standalone widget system and is independent of the window manager, which makes it suitable for Enlightenment. Rofi is available as an Arch package and works as an application launcher/window switcher. Nemo is also available in Arch Extra and is a good Windows-Explorer-like file manager candidate. ([Elkowar][1])

## Target dock layout

Bottom centre:

```text
[ Arch icon ] [ Search ] [ Favourite apps ... ] | [ Battery/Performance ] [ Network ] [ Sound ] [ Day Date Time ] [ Action Centre ]
```

No top panel.

The Arch icon and Search icon both open Rofi, but we can later make the Arch icon open a more Windows-like Start Menu while Search opens a search-only launcher.

# 5. Create the backend scripts

## Battery/performance widget

```bash
nano ~/.config/eww/scripts/power_status.sh
```

```bash
#!/bin/bash

BAT_PATH=$(find /sys/class/power_supply -maxdepth 1 -name "BAT*" | head -n 1)

if [ -n "$BAT_PATH" ]; then
    CAPACITY=$(cat "$BAT_PATH/capacity" 2>/dev/null)
    STATUS=$(cat "$BAT_PATH/status" 2>/dev/null)

    if [ "$STATUS" = "Charging" ]; then
        echo "󰂄 ${CAPACITY}%"
    else
        echo "󰁹 ${CAPACITY}%"
    fi
else
    LOAD=$(awk '{print int($1 * 100)}' /proc/loadavg)
    PROFILE=$(powerprofilesctl get 2>/dev/null || echo "balanced")
    echo "󰍛 ${LOAD}% ${PROFILE}"
fi
```

## Network speed widget

```bash
nano ~/.config/eww/scripts/network_status.sh
```

```bash
#!/bin/bash

IFACE=$(ip route | awk '/default/ {print $5; exit}')

if [ -z "$IFACE" ]; then
    echo "󰤮 Offline"
    exit 0
fi

RX1=$(cat /sys/class/net/"$IFACE"/statistics/rx_bytes)
TX1=$(cat /sys/class/net/"$IFACE"/statistics/tx_bytes)
sleep 1
RX2=$(cat /sys/class/net/"$IFACE"/statistics/rx_bytes)
TX2=$(cat /sys/class/net/"$IFACE"/statistics/tx_bytes)

RX_RATE=$(( (RX2 - RX1) / 1024 ))
TX_RATE=$(( (TX2 - TX1) / 1024 ))

echo "󰤨 ${RX_RATE}K↓ ${TX_RATE}K↑"
```

## Sound widget

```bash
nano ~/.config/eww/scripts/volume_status.sh
```

```bash
#!/bin/bash

if pamixer --get-mute | grep -q true; then
    echo "󰖁 Muted"
else
    VOL=$(pamixer --get-volume 2>/dev/null || echo 0)

    if [ "$VOL" -eq 0 ]; then
        echo "󰕿 ${VOL}%"
    elif [ "$VOL" -lt 50 ]; then
        echo "󰖀 ${VOL}%"
    else
        echo "󰕾 ${VOL}%"
    fi
fi
```

## Performance menu

This gives your desktop/laptop a starting version of the “5 custom performance modes” idea.

```bash
nano ~/.config/eww/scripts/performance_menu.sh
```

```bash
#!/bin/bash

CHOICE=$(printf "Eco\nBalanced\nPerformance\nQuiet\nTurbo" | rofi -dmenu -p "Performance Mode")

case "$CHOICE" in
    Eco)
        powerprofilesctl set power-saver 2>/dev/null
        sudo cpupower frequency-set -g powersave >/dev/null 2>&1
        notify-send "Performance Mode" "Eco mode enabled"
        ;;
    Balanced)
        powerprofilesctl set balanced 2>/dev/null
        sudo cpupower frequency-set -g schedutil >/dev/null 2>&1
        notify-send "Performance Mode" "Balanced mode enabled"
        ;;
    Performance)
        powerprofilesctl set performance 2>/dev/null
        sudo cpupower frequency-set -g performance >/dev/null 2>&1
        notify-send "Performance Mode" "Performance mode enabled"
        ;;
    Quiet)
        powerprofilesctl set power-saver 2>/dev/null
        sudo cpupower frequency-set -g powersave >/dev/null 2>&1
        notify-send "Performance Mode" "Quiet mode enabled"
        ;;
    Turbo)
        powerprofilesctl set performance 2>/dev/null
        sudo cpupower frequency-set -g performance >/dev/null 2>&1
        notify-send "Performance Mode" "Turbo mode enabled"
        ;;
esac
```

Make all scripts executable:


# 10. What still needs custom work later

This gives you the core dock, launcher, status widgets, action centre, and Super-key behaviour. The remaining advanced parts are:

1. **True running-app indicators** under favourite apps
   Eww can show fixed favourite apps easily, but showing live running windows cleanly requires extra scripts using `wmctrl` or `xdotool`.

2. **Proper custom 5 performance modes**
   The current script maps your 5 labels onto Linux governors and `powerprofilesctl`. Later we can make real profiles with CPU governor, brightness, turbo boost, fan control, GPU mode, and laptop/desktop detection.

3. **Windows-like action centre polish**
   The current action centre opens from the right side and gives quick actions. Later we can add Wi-Fi list, Bluetooth, night light, screenshot, notifications, and calendar.

4. **File manager final choice**
   I recommend **Nemo** first because it gives a more familiar Windows-like file manager experience than many ultra-minimal Linux file managers. If you want lighter, use Thunar instead. Thunar is also officially documented and installable on Arch. ([Arch Wiki][3])

Your next install-script addition should be this: install the packages above, create the Eww/Rofi/sxhkd configs in `/home/ibrahim`, and autostart them after Enlightenment login.
