One hard truth: the **GDM login screen and GNOME lock screen are not the same as your logged-in desktop shell**. Dash-to-Panel runs in your user session; it does not redesign GDM’s greeter. GDM login background changes are handled separately through GDM theming/settings, which Arch documents separately from normal GNOME Shell extension settings. ([Arch Wiki][2])

Also, **“Not listed?” does not create users**. It only allows manual login for an already-existing user account. The supported GDM setting can disable the user list, but if we do that, your visible `ibrahim` tile and profile picture disappear too. Keeping the profile tile means keeping GDM’s normal user-list behaviour. AccountsService is the mechanism GNOME/GDM uses for visible login users and profile icons; your previous script created that file, but this version rewrites it more directly and restarts the service. ([askubuntu.com][3])
