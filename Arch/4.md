You are right to push back. The Eww-on-Enlightenment path is not production-grade for the behaviour you want. The dock is being faked by repeatedly opening/closing a widget window, so it will never feel like Ubuntu/macOS: no proper compositor-managed slide, weak hover behaviour, weak icon handling, and unreliable desktop integration.

The correct fix is to stop trying to turn Enlightenment into Ubuntu. Use **Arch as the OS** and **GNOME Shell as the UI layer**. GNOME gives you the Ubuntu-style shell, native animations, native tooltips, app indicators, proper window focus behaviour, GNOME Files/Nautilus, GNOME Terminal, extension-based dock customisation, and terminal-configurable keybindings. Arch still remains Arch: pacman, AUR, lightweight base install, your own package choices. Arch officially packages GNOME Shell, Nautilus, Desktop Icons NG, ArcMenu, AppIndicator support, Dash-to-Panel, Extension Manager, and Ubuntu fonts; Dash-to-Dock/Ubuntu Dock style is available through GNOME extensions/AUR. ([Arch Linux][1])

Important: Ubuntu Dock on extensions.gnome.org is only a placeholder/mock extension and tells users to use Dash-to-Dock for the same functionality. So the practical Arch equivalent is **Dash-to-Dock**, not the fake “Ubuntu Dock” listing. ([GNOME Extensions][2])

from gi.repository import Nautilus, GObject
import os
import shutil
import subprocess


class IBContextTools(GObject.GObject, Nautilus.MenuProvider):
    def _path_from_file(self, file_obj):
        try:
            location = file_obj.get_location()
            if location is None:
                return os.path.expanduser("~")
            path = location.get_path()
            if path is None:
                return os.path.expanduser("~")
            return path
        except Exception:
            return os.path.expanduser("~")

    def _directory_for_selection(self, files):
        if not files:
            return os.path.expanduser("~")

        path = self._path_from_file(files[0])

        if os.path.isdir(path):
            return path

        return os.path.dirname(path)

    def _open_terminal(self, menu, path):
        terminal = shutil.which("gnome-terminal")

        if terminal:
            subprocess.Popen([terminal, "--working-directory", path])
        else:
            subprocess.Popen(["sh", "-lc", "x-terminal-emulator"], cwd=path)

    def _open_code(self, menu, path):
        code = shutil.which("code") or shutil.which("codium")

        if code:
            subprocess.Popen([code, path])
        else:
            subprocess.Popen(["notify-send", "Open in Code", "VS Code/Codium command was not found."])

    def _new_text_file(self, menu, path):
        base = "New Text File"
        candidate = os.path.join(path, base + ".txt")
        i = 1

        while os.path.exists(candidate):
            candidate = os.path.join(path, f"{base} {i}.txt")
            i += 1

        open(candidate, "w").close()

    def _items_for_path(self, path):
        terminal_item = Nautilus.MenuItem(
            name="IBContextTools::OpenInTerminal",
            label="Open in Terminal",
            tip="Open GNOME Terminal in this folder",
            icon="utilities-terminal"
        )
        terminal_item.connect("activate", self._open_terminal, path)

        code_item = Nautilus.MenuItem(
            name="IBContextTools::OpenInCode",
            label="Open in Code",
            tip="Open this folder in Visual Studio Code",
            icon="visual-studio-code"
        )
        code_item.connect("activate", self._open_code, path)

        new_file_item = Nautilus.MenuItem(
            name="IBContextTools::NewTextFile",
            label="New File",
            tip="Create a new empty text file",
            icon="text-x-generic"
        )
        new_file_item.connect("activate", self._new_text_file, path)

        return [new_file_item, terminal_item, code_item]

    def get_background_items(self, current_folder):
        path = self._path_from_file(current_folder)
        return self._items_for_path(path)

    def get_file_items(self, files):
        path = self._directory_for_selection(files)
        return self._items_for_path(path)
PY

#!/bin/bash

set -euo pipefail

VALUE="${1:-}"

if [ -z "$VALUE" ]; then
    echo "Usage: ib-brightness 0.30 to 1.00"
    exit 1
fi

if command -v brightnessctl >/dev/null 2>&1 && brightnessctl -m >/dev/null 2>&1; then
    PERCENT="$(awk -v v="$VALUE" 'BEGIN { printf "%d%%", v * 100 }')"
    brightnessctl set "$PERCENT"
    exit 0
fi

if command -v xrandr >/dev/null 2>&1; then
    OUTPUT="$(xrandr --current | awk '/ connected/ {print $1; exit}')"

    if [ -n "$OUTPUT" ]; then
        xrandr --output "$OUTPUT" --brightness "$VALUE"
        exit 0
    fi

echo "No controllable backlight or xrandr output found."
exit 1
EOF

GNOME absolutely has enough customisation power for what you want. It is not as endlessly free-form as KDE, but with GNOME extensions, `gsettings`, `dconf`, shell themes, Nautilus extensions, and AUR packages, you can build an Ubuntu/macOS/Windows hybrid without fighting the window manager. Dash-to-Dock is specifically built to turn GNOME’s dash into a real dock for launching and switching apps, with side/bottom placement. Desktop Icons NG restores desktop icon behaviour in GNOME Shell and works with Dash-to-Dock/Dash-to-Panel. ([GNOME Extensions][3])

One important limitation: the **desktop background right-click menu** in GNOME is controlled by Desktop Icons NG, not Nautilus itself. The script gives you reliable **New File**, **Open in Terminal**, and **Open in Code** inside Nautilus via Nautilus Python, and it creates the GNOME template needed for “New Document/New File” behaviour. Nautilus Python is officially packaged by Arch as bindings for the Nautilus extension API. For the desktop background itself, we can later write a dedicated GNOME Shell extension if DING does not expose every context-menu item you want. ([Arch Linux][4])

For brightness: the screen dimming you saw before sleep can be DPMS/screen blanking rather than real backlight brightness. The script adds `ib-brightness`, which first tries real hardware brightness through `brightnessctl`; if VirtualBox exposes no backlight, it falls back to Xorg gamma/output brightness through `xrandr`.
