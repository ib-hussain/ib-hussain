Understood. Do **not** run the previous `gnome-rice-v6.sh`. We are going to split the fix into **four smaller files** so each layer is easier to verify.

The important correction is this: your current logs show `xrandr` is running under **Wayland**, so `xcape`, `xrandr brightness`, and some screenshot behaviour will not behave correctly until the session is forced to **GNOME on Xorg**. GDM can be configured to use Xorg by setting `WaylandEnable=false` and `DefaultSession=gnome-xorg.desktop`; AccountsService can also store the user’s preferred session through `XSession=gnome-xorg`. ([Fedora Docs][1])

Dash-to-Panel remains the correct base because it is designed to combine launchers, running apps, and system tray indicators into one bottom panel. ([GNOME Extensions][2]) The major mistake in earlier scripts was that some Dash-to-Panel JSON settings were passed without proper string quoting, so keys like `panel-lengths`, `panel-anchors`, and `panel-element-positions` did not reliably apply.
